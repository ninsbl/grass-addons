DESCRIPTION

The i.jaxa.download addon allows exploring and downloading satellite products from the G-Portal of the Japan Aerospace Exploration Agency (JAXA). The G-Portal provides data via FTP that is organized in a hirachical folder structure with the following levels that are reflected in the module options:
  1. product_type
  2. spacecraft
  3. sensor
  4. product
  5. version
All the above options need to be provided in order to download data from the portal. If no input is provided for any of these options, the available content of the respective hirarchy is printed. The elements from the returned list can then be used as input to the missing option. In addition, the following options can be used to further filter data products to download. They correspond to elements in the file names.
  • time_unit
  • projection
  • resolution
  • tiles
Not all combinations of the available options and filters are valid and lead to actually downloadable data. Predefined products represent combinations of option values and filters that represent specific products. If the predefined_products is used, other options and filters are overwritten (except for the tiles option).

Returned data can in addition be filtered using the start and end options.

The module supports parallel download if instructed to use more than one core in the nprocs option.

Downloads are resumed if incomplete files are in the local directory.

To connect to the G-Portal a user name is required. see Register new account page for signing up.

i.jaxa.download reads the user credentials from a settings file. That file must contain the user name in the first line.

The user name can be also defined interactively when settings=- is given. Note that interactive prompt does not work in the graphical user interface. 

myusername

NOTES

The module is primarily written for data products from the GCOM-C spacecraft. Other data products can be explored and downloaded as well, but that is less tested and may require adjustment of the code.

The module does not provide any spatial filtering of the data and the user has to identify e.g. relevant tiles.

The output directory is created if not yet available.

tiles=0217,0218,0317,0318
